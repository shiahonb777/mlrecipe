mlrecipe

github.com/shiahonb777/mlrecipe · MIT

A content-addressed format for LoRA fine-tunes. The recipe — a small TOML file plus a hashed adapter — points at a base model on Hugging Face and rebuilds a merged checkpoint that is bit-identical to peft.merge_and_unload(). No new servers, no new accounts. GitHub releases host everything.

This page indexes any GitHub repository that publishes a recipe. Browse a known one, search the public ones, or copy the three commands that publish your own.

You have a trained LoRA adapter directory — anything PEFT writes via save_pretrained. Three commands turn it into a recipe and push it as a GitHub release. The adapter never leaves your machine before that git push.

  1. Install mlrecipe

    One-time, from the CLI. Python 3.9+.

    $pip install git+https://github.com/shiahonb777/mlrecipe.git
  2. Convert your adapter into a recipe repo

    Point from-peft at your adapter directory and a fresh repo path. The adapter file (typically a few MB) gets stored content-addressed; the recipe itself is a hundred or so bytes of TOML.

    $mlrecipe from-peft \
    ./my-adapter \
    ./my-recipe-repo \
    --base-ref Qwen/Qwen2.5-1.5B
  3. Push the repo and tag a release

    The recipe directory is a regular Git repository. Push it to GitHub and create a release; the .recipe-bundle-v1.tar.gz attached to that release is what this page reads.

    $cd my-recipe-repo
$git init && git add . && git commit -m "Initial recipe"
$gh repo create my-recipe-repo --public --push
$mlrecipe push v1   # bundles + creates the release
  4. Share the link

    Once the release exists, anyone with this URL pattern sees your recipe details rendered. Replace the example in the URL bar with your own user/repo@tag:

    →mlrecipe.shiahonb777.dev/?ref=<your-user>/<repo>@v1

How a recipe is verified

Every recipe references its adapter by a content hash (sha256:...) and its base model by a Hugging Face revision. mlrecipe materialize downloads the base, downloads the adapter, calls peft.merge_and_unload(), and writes the merged safetensors. On the GPT-2 and Qwen 2.5 examples on this page, all 148 of 148 tensors match the PEFT reference output to the bit; the maximum element-wise difference is zero.

The shipped recipe and the merged checkpoint are interchangeable for inference, but the recipe is two to three orders of magnitude smaller. That ratio is what makes recipes practical to host on a free tier.

A recipe is a TOML file plus a content-addressed adapter. Verification is local; distribution is GitHub.
