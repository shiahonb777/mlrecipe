The plan: download the base model and the recipe's LoRA adapter, fold the LoRA into the base in JavaScript, and verify the result has the same hashes the Python peft.merge_and_unload() path produces. Browser-side. No upload.

Not yet built. The browser materialize path runs against GPT-2 sized bases (≤500 MB) only, and is prototyped against shiahonb777/gpt2-alpaca-recipe@v1. Until it ships, use the CLI: 
$ pip install git+https://github.com/shiahonb777/mlrecipe.git
$ mlrecipe clone shiahonb777/gpt2-alpaca-recipe@v1
$ cd gpt2-alpaca-recipe
$ mlrecipe materialize ./merged

For larger models — Qwen 1.5B and up — the CLI is the right tool regardless. Browser memory caps out around 2 GB.
